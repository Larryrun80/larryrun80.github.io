Hexo on Github(1): 创建和发布你的Github Blog

本文将帮助你以当前最简单的方法在Github搭建你的个人博客，Hexo是一个简单、快速、强大的框架，它将帮助我们完成博客文章的创建、删除和更新。
Hexo基于Node.js， 支持Markdown格式，如果你对这两个名词感到陌生也没关系，只需十分钟你就可以拥有自己的Github博客体系了。
本文主要基于MacOS进行介绍，如果你的系统是Windows和Linux，也没有关系，整个流程是大同小异的。

开始之前

你的电脑中需要已经安装好 Node.js 和 Git。
Hexo基于Node.js并且你需要通过Git将你的本地博客提交到Github。
如果你不确定你是否已经安装了Node.js或Git，可以先不管他，跟着我流程往下走。

建立你的Github账号

打开Github， 用自己的邮箱注册即可
Github 为每个用户提供了一个 pages 项目，你可以通过pages访问属于你自己的静态website服务
Github Pages 分为两种：
1 归属于用户或组织的全局Page
新建一个Github项目，并以 yourUserName.github.io 命名，整个账户只有一个，必须push到项目的master分支
yourUserName是你的Github账号
2 归属于项目的Page
这种Page归属于每一个项目，可以把静态页面push到gh-pages分支
通常我们使用第一种作为个人博客，我没有试过第二种，但理论上也是可行的。

Hexo目录结构

_config.yml是整个项目的配置文件，Hexo new命令创建的博客就放在source文件夹中，themes用于放置你的博客模板

安装Node.js

推荐使用 nvm 安装Node.js，注意当前nvm并不支持Windows，在官网他们给出了两个替代方案。
如果你想更简单一些，打开 Node.js官网 并直接下载安装包安装。

安装Git

-Windows: Download & install msysgit.
-Mac: Git应该已经是Mac的自带组件了，你可以通过 installer 或 brew 安装（brew install git）
-Linux (Ubuntu, Debian): sudo apt-get install git-core
-Linux (Fedora, Red Hat, CentOS): sudo yum install git-core
你也可以访问 Git官网 完成整个安装过程。
